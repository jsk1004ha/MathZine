# MathZine

MathZine은 뉴스 사이트 문법, 특히 뉴욕타임즈식 레이아웃을 참고해 만든 수학 잡지 플랫폼입니다. 기사 읽기는 비회원에게도 열려 있고, 리로스쿨 인증을 거친 사용자는 닉네임으로 게시판, 댓글, 명예의 전당 풀이 제출에 참여할 수 있습니다. 기사는 먼저 제출되고, 어드민이 호수 단위로 공개합니다.

## 주요 특징

- `localhost:3000` 하나로 프론트엔드와 백엔드를 함께 실행하는 Next.js 단일 앱
- PPT에서 확인한 `최신호 / 명예의 전당 / 인기 기사 / 게시판` 포털형 메인페이지
- 얇은 구분선, 흑백 중심 팔레트, 세리프 헤드라인을 사용하는 뉴욕타임즈형 에디토리얼 UI
- 블랙레터 계열 서체를 사용하는 `MathZine` 마스트헤드
- 기사 상세 페이지, 댓글, 좋아요, 조회수, 문제 풀이 기반 명예의 전당 랭킹
- `/issues` 와 `/issues/[issueSlug]` 에서 잡지 호별 아카이브 탐색
- 각 호 페이지에서 PDF 묶음 다운로드 지원
- 호별 상세는 저장된 `issueSlug`와 표시용 호수명을 함께 정규화해 기존 데이터가 섞여 있어도 안정적으로 진입
- 한글이 포함된 호수/기사 slug는 상세 페이지 진입 시 명시적으로 디코드해 `1호`, `2026년 6월호` 같은 경로도 안정적으로 처리
- 기사 상세 페이지에 연결된 문제를 두고 PDF 풀이를 바로 제출하는 흐름
- 블록 기반 기사 편집기: 문단, 라틴체 문단, 콜아웃, 정리, 증명, 수식, 코드, 사진, 영상, 링크, 참고문헌
- 클릭형 수식 템플릿, 인라인 서식 툴바, 자동 URL 링크 인식, 라이브 프리뷰를 포함한 시각적 기사 작성 스튜디오
- 블록에 연결된 각주와 참고문헌 목록을 기사 하단에 자동 생성
- 로그인, 게시판, 어드민, 기사 제출기 폼을 같은 기준의 간격과 정렬로 정리한 읽기 중심 UI
- 로그인 없이 기사 열람 가능
- 리로스쿨 학생 계정 + 어드민이 직접 만드는 local 교사 계정 구조
- `member / reporter / teacher / admin` 권한 모델과 자동 로그인 유지 옵션
- 모바일 반응형 레이아웃
- SQLite 저장소와 최소 권한 세션 쿠키를 사용하는 간단한 CMS 백엔드
- 어드민 화면을 `대시보드 / 편집 관리 / 계정 관리 / 명예의 전당`으로 분리하고, 제출 기사 미리보기, teacher 계정 생성, 호수 공개, 기사 연결형 명예의 전당 문제 등록, PDF 풀이 점수 채점 지원

## 기술 스택

- Next.js 16 App Router
- React 19
- SQLite 데이터베이스 (`storage/mathzine.db`)
- 서버 측 리로스쿨 인증 연동 또는 로컬 데모 모드

## 실행 방법

```bash
npm install
npm run dev
```

개발 서버는 기본적으로 [http://localhost:3000](http://localhost:3000) 에서 열립니다.

프로덕션 빌드:

```bash
npm run build
npm start
```

## 리로스쿨 인증 모드

기본값은 로컬 데모 모드입니다. 데모 모드에서는 임의의 리로스쿨 ID/비밀번호를 넣어 계정 흐름을 검증할 수 있습니다.

실제 리로스쿨과 연동하려면 환경 변수를 설정하세요.

```bash
cp .env.example .env.local
# 이후 .env.local 에서 mock -> live 로 변경
```

설명:

- `RIRO_MODE=live`: 실제 리로스쿨 로그인 절차를 사용합니다.
- `RIRO_BASE_URL`: 참고한 `riroschoolauth.py`와 동일한 방식으로 학교 리로스쿨 도메인을 지정합니다.

## 보안 설계

- 리로스쿨 비밀번호는 저장하지 않습니다.
- local 계정 비밀번호는 `scrypt` 해시로 저장합니다.
- 세션 쿠키는 `HttpOnly`, `SameSite=Lax`, 프로덕션에서 `Secure` 옵션으로 발급합니다.
- 쓰기 API는 동일 출처 요청만 허용합니다.
- 입력값은 서버에서 길이 제한과 정규화 과정을 거칩니다.
- 기사 읽기는 공개, 회원 행동은 닉네임 중심으로 표시됩니다.
- 기사 제출은 `reporter` 또는 `admin`, 호수 공개/권한 변경/직접 계정 생성은 `admin`으로 제한됩니다.
- 로컬 DB는 SQLite WAL 모드로 동작합니다.

## 주요 경로

- `/` : 메인페이지
- `/issues` : 호별 아카이브
- `/issues/[issueSlug]` : 특정 호 기사 묶음
- `/articles/[slug]` : 기사 상세
- `/hall-of-fame` : 명예의 전당
- `/board` : 게시판
- `/login` : 로그인
- `/signup` : 회원가입
- `/studio` : 기사 작성 스튜디오
- `/profile` : 내 정보
- `/admin` : 어드민 운영 화면
- `/admin/editorial` : 제출 기사와 호수 공개 관리
- `/admin/accounts` : 역할 변경과 직접 계정 생성
- `/admin/hall-of-fame` : 문제 등록과 풀이 채점

## 기사 작성기

`/studio`는 기자 계정용 블록 편집기입니다.

- 문단 스타일: 본문, 리드, 라틴체, 주석 톤
- 구조 블록: 소제목, 콜아웃, 정리(Theorem), 증명(Proof), 코드, 링크 카드
- 수식 블록: 등식, 분수, 지수, 제곱근, 시그마, 적분, 직접 입력 템플릿
- 미디어 블록: 이미지 업로드/URL, 영상 업로드/URL, 포스터 이미지
- 인라인 서식 툴바: 굵게, 기울임, 밑줄, 취소선, 인용, 코드, 링크, 서식 제거
- 하이퍼링크: 문단 안에 URL을 그대로 입력하면 자동 링크로 렌더링되고, 필요하면 툴바로 선택 영역 링크 삽입 가능
- 참고문헌: 블록별 각주 연결과 기사 하단 참고문헌 자동 정리
- 라이브 프리뷰: 저장 전 기사 지면을 우측 패널에서 바로 확인
- 저장 방식: 작성 즉시 공개되지 않고 어드민 검토용으로 제출되며, 공개 전에는 작성자와 어드민만 미리 볼 수 있음
- 기사 상세 반응 패널: 좋아요, 조회수, 댓글 수, 다른 볼거리를 본문 아래 댓글 위에 배치

## 저장소 구조

- `app/` : 페이지와 API 라우트
- `components/` : UI 컴포넌트
- `lib/` : 저장소, 인증, 시드 데이터, 보안 유틸리티
- `lib/` : 저장소, 인증, 기사 문서 블록, 보안 유틸리티
- `storage/mathzine.db` : 런타임 SQLite 데이터베이스
- `storage/uploads/` : 문제 풀이 PDF 및 기사 미디어 업로드 위치

기존 JSON 저장 파일은 제거했습니다. 현재 런타임 읽기/쓰기는 `storage/mathzine.db`만 사용합니다.
`storage/mathzine.db`가 없으면 서버 실행이나 프로덕션 빌드 과정에서 테이블이 자동 생성됩니다. 기본 상태는 빈 데이터베이스입니다.
샘플 기사나 샘플 문제는 자동으로 넣지 않습니다.

## 로컬 백엔드 추천

현재 구현은 이미 `Next.js Route Handlers + SQLite (better-sqlite3)` 조합을 사용합니다.

- 이유: 서버 하나로 끝나고, 설치가 가볍고, 파일 하나로 백업/이동이 쉬우며, 권한/랭킹/게시판처럼 관계형 데이터에 잘 맞습니다.
- 로컬 환경에서는 별도 DB 서버 없이 바로 운용할 수 있고, 업로드 파일과 함께 백업하기 쉽습니다.

## 참고 사항

- Next.js 16의 Turbopack 빌드에서 파일 기반 저장소 추적으로 인한 경고가 1건 남아 있을 수 있습니다. 현재 빌드는 통과하며 동작에는 영향이 없습니다.
- `MathZine` 로고는 Windows 환경에서 `Old English Text MT`가 있으면 이를 우선 사용하고, 없으면 블랙레터 계열 폴백 서체로 렌더링됩니다.
