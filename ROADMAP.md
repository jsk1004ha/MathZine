# MathZine Roadmap

MathZine 운영 안정성, 확장성, 사용자 경험 개선을 위한 남은 과제 정리입니다.

`0.7.1` 기준으로 Rate Limit, 보안 헤더, 업로드 검증, 공통 에러 스키마, 헬스체크, 감사 로그, 사용자 차단, 검색 결과 페이지, 게시판/아카이브 필터, 마이페이지 활동 대시보드, 기사 2단 컬럼 지면, 기본 CI/quality gate는 반영했습니다.

## 1. 보안 및 운영 안정성

### 운영 관측성 보강

- 장애 분석용 기본 메트릭 수집 체계 마련

### 사용자 차단 및 밴 기능

- 차단 사유 기반 이력 조회와 해제 이력 추적
- 게시판/댓글/좋아요 단위의 세부 제한 정책 추가

## 2. 품질 관리 및 개발 프로세스

### 테스트 및 품질 게이트

- 더 깊은 통합 테스트와 회귀 테스트 확장
- JS 기반 빌드 검증 외 별도 타입 안정성 전략 정리

## 3. 데이터 구조 및 확장성

### 저장 구조 정규화

- 현재 blob 저장 구조에서 핵심 도메인부터 정규화 테이블로 단계적 전환

### 목록 API 페이지네이션

- 추가 엔드포인트에도 동일한 페이지네이션/정렬 기준 확장

### Article 데이터 로딩 최적화

- 메타데이터와 본문 조회 중복 제거
- 관련 기사 계산을 위한 전체 목록 재로딩 구조 개선

## 4. SEO 및 배포 실무 기능

### 검색 유입 대응 강화

- 기사별 대표 이미지/소셜 미리보기까지 포함한 SEO 보강

## 5. API 설계 및 에러 처리

### 공통 에러 계약 정의

- 상태코드, 에러코드, 메시지를 포함한 공통 에러 스키마 정의
- API 응답 계약 표준화

## 6. 사용자 경험 개선

### 인증 폼 실시간 피드백

- 더 세밀한 필드별 상태 메시지와 접근성 점검

### 댓글/좋아요/제출 성공 피드백

- 글로벌 토스트 시스템 도입 여부 검토

### 검색 결과 페이지 고도화

- 하이라이트, 추천 검색어, 결과 없음 안내 고도화

### 마이페이지 강화

- 활동 통계와 상태별 필터링 고도화

### 관련 콘텐츠 추천 개선

- 텍스트 유사도 기반 추천 정밀화

### 앱 내부 온보딩

- 실제 단계별 온보딩 UI와 도움말 배치

### 계정 편의 기능

- 복구 채널이 있는 비밀번호 재설정 흐름 설계
